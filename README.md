TextFrequencyAnalyzer
TextFrequencyAnalyzer — консольное приложение на C++ для генерации, валидации и частотного анализа JSON‑файлов с текстами.

Программа позволяет:

Генерировать JSON‑файлы с текстами и стоп‑словами.

Валидировать структуру данных (text, stopwords).

Выполнять частотный анализ: общее число слов, число уникальных слов, top‑N слов.

Нормализовать регистр, отбрасывать стоп‑слова, работать с пунктуацией.

Измерять время выполнения анализа (benchmarks) на большом числе файлов.

Запуск программы
Скачай репозиторий и собери проект (см. раздел «Сборка» в README).

Запусти скомпилированный файл, например TextFrequencyAnalyzer.exe (Windows) или ./TextFrequencyAnalyzer (Linux).

После запуска появится меню:

================ ЧАСТОТНЫЙ АНАЛИЗ ТЕКСТА ================
Выберите действие
1) Проверка валидности и анализ файлов
2) Режим отладки (генерация и бенчмарк)
3) Выход

Для анализа данных положи свои JSON‑файлы text_0.json, text_1.json, … в рабочую папку рядом с исполняемым файлом. Примеры JSON находятся в каталоге data/.

Формат JSON
Минимальный корректный файл:

json
{
  "text": "В начале было Слово. И Слово было у Бога...",
  "stopwords": ["и", "в", "у"]
}
text — обязательная строка с текстом для анализа.

stopwords — необязательный массив строк со стоп‑словами.

Пустые и битые JSON‑файлы программа пропускает с сообщением об ошибке.

Демонстрационный сценарий
1. Генерация JSON‑файлов (Режим отладки)
Выбираем 2) Режим отладки.

Выбираем 1) Сгенерировать JSON-файлы.

Вводим количество файлов, например 5.

Создаются файлы:

text
text_0.json
text_1.json
...
text_4.json
Каждый содержит поле text и список stopwords.

2. Валидация и частотный анализ
Выбираем 1) Проверка валидности и анализ файлов.

Вводим количество файлов для проверки, например 5.

Для каждого файла программа:

проверяет структуру JSON (text — строка, stopwords — массив строк);

выполняет частотный анализ текста;

выводит статистику и top‑слова.

Пример вывода:

text
Проверка text_0.json
Анализ text_0.json
Всего слов: 42
Уникальных: 25
Top слова:
слово : 4
текст : 3
анализ : 2
-----------------------
3. Замер времени (бенчмарк)
Выбираем 2) Режим отладки.

Выбираем 2) Замер времени анализа.

Вводим количество файлов (например, 1000).

Программа последовательно анализирует все файлы и выводит суммарное время обработки:

text
===== ЗАМЕР ВРЕМЕНИ =====
Файлов проверено: 1000
Общее время: 1234 мс
